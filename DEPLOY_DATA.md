# تعليمات نقل البيانات إلى سيرفر الإنتاج
## Data Deployment Instructions

لقد تم إنشاء ملف **`production_data.sql`** يحتوي على جميع البيانات الحالية (ما عدا المستخدمين) وجاهز للرفع.

### 1. طريقة الرفع (عبر Git)
بما أن الملف موجود ضمن مستودع Git، يمكنك سحبه مباشرة على السيرفر:

```bash
# على سيرفر الإنتاج
cd /path/to/project
git pull origin main
```

### 2. طريقة الاستيراد (Import)
بعد سحب الملف، قم بتنفيذ الأمر التالي لإدخال البيانات إلى قاعدة البيانات.
**ملاحظة:** يجب استبدال `<db_user>` و `<db_name>` و `<host>` بالبيانات الصحيحة للسيرفر.

```bash
# أمر الاستيراد (PostgreSQL)
# إذا كان السيرفر محلي:
psql -U postgres -d garage_db -f production_data.sql

# إذا كان السيرفر خارجي (مثل PythonAnywhere):
psql -h <host_address> -p <port> -U <username> -d <database_name> -f production_data.sql
```

### ماذا يفعل هذا الملف؟
1. **يتخطى التحقق من المفاتيح الأجنبية مؤقتاً**: لضمان عدم حدوث أخطاء بسبب ترتيب الجداول أو غياب بعض المستخدمين.
2. **يتجاهل البيانات المكررة**: إذا كان هناك سجل موجود مسبقاً (مثل الأدوار أو الإعدادات)، سيتم تجاهله ولن يتكرر (`ON CONFLICT DO NOTHING`).
3. **يستثني المستخدمين**: لن يتم تعديل أو حذف أي مستخدمين موجودين على السيرفر، ولن يتم نقل مستخدمي النسخة المحلية.

### حل مشاكل محتملة
- **خطأ في الصلاحيات**: إذا ظهر خطأ `permission denied to set parameter "session_replication_role"`, فهذا يعني أن مستخدم قاعدة البيانات ليس لديه صلاحيات Superuser. في هذه الحالة، يجب تنفيذ الأمر بحساب `postgres` أو مسؤول النظام.
- **مستخدمين مفقودين**: بما أننا استثنينا المستخدمين، قد تظهر بعض السجلات (مثل المبيعات) مرتبطة بمستخدمين غير موجودين. هذا لن يوقف النظام ولكنه قد يظهر "Unknown User" في بعض التقارير.
